Step 4: GitHub OAuth Configuration

Set up GitHub OAuth for secure authentication with your Master MCP Server:

Important Security Note

Never commit your client secret to version control. Store it securely as an environment variable.

1. Create GitHub OAuth App

  1. 1

    Go to GitHub Developer Settings

  2. 2

    Click "New OAuth App"

  3. 3

    Fill in the application details:

    Application name: Master MCP Server
    Homepage URL: http://localhost:3005
    Authorization callback URL: http://localhost:3005/oauth/callback
  4. 4

    Copy your Client ID and generate a Client Secret

2. Configure Environment Variables

Create a .env file in your project root:

# GitHub OAuth Configuration
GITHUB_CLIENT_ID=your_client_id_here
GITHUB_CLIENT_SECRET=your_client_secret_here

# Server Configuration
PORT=3005
BASE_URL=http://localhost:3005

3. Update Configuration File

Your config.json should look like this:

{
  "hosting": {
    "platform": "node",
    "port": 3005,
    "base_url": "http://localhost:3005"
  },
  "master_oauth": {
    "authorization_endpoint": "https://github.com/login/oauth/authorize",
    "token_endpoint": "https://github.com/login/oauth/access_token",
    "client_id": "your_client_id_here",
    "client_secret": "your_client_secret_here",
    "redirect_uri": "http://localhost:3005/oauth/callback",
    "scopes": ["read:user", "user:email"]
  },
  "servers": []
}

Step 5: Server Configuration

Configure your Master MCP Server with the proper settings and security:

Configuration Options

üè† Hosting Configuration
  • platform: node | cloudflare-workers | koyeb | docker
  • port: Server port (default: 3000)
  • base_url: Public URL for callbacks
üîê Authentication
  • authorization_endpoint: OAuth auth URL
  • token_endpoint: OAuth token URL
  • scopes: Requested permissions
‚ö° Advanced Features
  • Load Balancing: round_robin | weighted | health
  • Circuit Breaker: Failure protection
  • Retry Policies: Automatic retry logic
üìä Monitoring
  • Health Checks: Server status monitoring
  • Metrics: Performance tracking
  • Logging: debug | info | warn | error

Environment-Based Configuration

You can override config values using environment variables:

# Override hosting settings
MASTER_MCP_HOSTING_PORT=3005
MASTER_MCP_HOSTING_BASE_URL=http://localhost:3005

# Override OAuth settings
MASTER_MCP_MASTER_OAUTH_CLIENT_ID=your_client_id
MASTER_MCP_MASTER_OAUTH_CLIENT_SECRET=your_secret

# Logging level
MASTER_MCP_LOGGING_LEVEL=debug

Validate Configuration

Validates your configuration against the JSON schema.

Step 7: Claude Desktop Integration

Connect your Master MCP Server to Claude Desktop for seamless AI interactions:

What is Claude Desktop?

Claude Desktop is a desktop application that allows you to interact with Claude AI while having access to your local files, tools, and MCP servers for enhanced functionality.

1. Install Claude Desktop

üçé macOS
Download for macOS

Requires macOS 11.0 or later

ü™ü Windows
Download for Windows

Requires Windows 10 or later

2. Configure Claude Desktop

Add your Master MCP Server to Claude Desktop's configuration:

Find Configuration File

macOS Location:

~/Library/Application Support/Claude/claude_desktop_config.json

Windows Location:

%APPDATA%\Claude\claude_desktop_config.json
Configuration Content
{
  "mcpServers": {
    "master-mcp-server": {
      "command": "node",
      "args": [
        "path/to/master-mcp-server/dist/index.js"
      ],
      "env": {
        "MASTER_CONFIG_PATH": "path/to/master-mcp-server/config.json"
      }
    }
  }
}

Replace path/to/master-mcp-server with your actual project path

3. Alternative: Direct Protocol Connection

For development or advanced use, connect directly via stdio:

{
  "mcpServers": {
    "master-mcp-server": {
      "command": "npm",
      "args": ["run", "mcp"],
      "cwd": "path/to/master-mcp-server"
    }
  }
}

Step 8: Adding Backend MCP Servers

Configure your Master MCP Server to manage and route to multiple backend MCP servers:

Server Aggregation

Your Master MCP Server acts as a central hub, aggregating capabilities from multiple backend servers and presenting them as a unified interface to Claude.

1. Server Configuration Structure

Add servers to your config.json file:

{
  "hosting": { /* ... */ },
  "master_oauth": { /* ... */ },
  "servers": [
    {
      "id": "filesystem-server",
      "type": "npm",
      "package": "@modelcontextprotocol/server-filesystem",
      "version": "latest",
      "auth_strategy": "bypass_auth",
      "config": {
        "environment": {
          "ALLOWED_DIRECTORIES": "/Users/username/Documents"
        },
        "args": ["--port", "3001"]
      }
    },
    {
      "id": "git-server",
      "type": "git",
      "url": "https://github.com/modelcontextprotocol/server-git.git",
      "branch": "main",
      "auth_strategy": "master_oauth",
      "config": {
        "environment": {
          "GIT_REPO_PATH": "/path/to/repo"
        }
      }
    }
  ]
}

2. Server Types & Sources

üì¶ NPM Packages
